MongoDB 为什么先暴跌、又暴涨？重新检查 MongoDB 与 Snowflake 的 AI 时代论点

MongoDB 这次大涨，核心不是市场突然忘记了之前的风险，而是 Q1 FY2027 把三月 Q4 财报后的主要担忧逐项修复了：Atlas 没有断档，全年指引上修，RPO 和自由现金流很强，AI / agent 叙事也从长期故事变成了早期可见 traction。

我的更新判断是：此前对 MongoDB 的短期 bearish 论点需要降级，但长期结构性问题没有消失。MongoDB 在 AI app starter 默认性上仍受 Postgres / Supabase / pgvector 压力，但它正在争夺 production agent backend 的默认权。

核心判断

  1. 三月暴跌主要是预期和指引问题。 MongoDB Q4 FY2026 当季结果不错，但 FY2027 初始收入指引显示增长降档，市场担心 Atlas 可见度、AI 收入兑现和软件股估值压力。
  2. 这次暴涨是典型 beat-and-raise。 Q1 FY2027 收入 6.876 亿美元，同比增长 25%；Atlas 收入同比增长超过 29%；RPO 同比增长 88%；公司上调 FY2027 revenue guide 到 29.2 亿至 29.6 亿美元。
  3. 不能再说 MongoDB 正被 AI 持续伤害。 Q1 数据反证很强：核心业务稳住，Atlas 仍强，AI / agentic workload 也开始出现更具体的早期信号。
  4. 但 Postgres 默认性压力仍然存在。 Postgres / Supabase / pgvector 在 AI app starter、MVP、RAG metadata、关系权限和 billing 场景里仍是非常强的默认起点。
  5. Snowflake 的 AI 货币化证据仍更直接。 Snowflake 把 AI adoption 更直接地映射到 product revenue、RPO、NRR 和全年指引；MongoDB 的 AI upside 更偏 agent memory、state、operational backend 和 production agent workloads。

三月财报后为什么暴跌

MongoDB Q4 FY2026 的当季结果本身并不差：总收入 6.951 亿美元，同比增长 27%；FY2026 全年收入 24.6 亿美元，同比增长 23%；Atlas 当季和全年收入均同比增长 29%。来源：MongoDB FY2026 Q4 SEC exhibit。

问题在 forward guide。Q4 后 MongoDB 给出的 FY2027 revenue guide 是 28.60 亿至 29.00 亿美元；Q1 FY2027 revenue guide 是 6.59 亿至 6.64 亿美元。来源：MongoDB FY2026 Q4 SEC exhibit。这意味着市场看到的是：当季 +27%、Atlas +29%，但全年增长预期切到约 16%-18%。

Investopedia 当时也把下跌归因于利润指引让市场失望，而不是当季 miss。文章称 MDB 当时下跌近 21%，并指出 Q4 revenue 6.95 亿美元和 adjusted EPS 1.65 美元都高于预期，但 Q1 adjusted EPS 指引 1.15-1.19 美元低于 Visible Alpha 共识 1.21 美元。来源：Investopedia。

所以三月暴跌的准确解释是：强当季、弱未来、软件 AI 焦虑、Atlas 可见度折价。 这不是数据库选型已经发生大规模迁移的证据。

这次为什么又暴涨

MongoDB Q1 FY2027 官方财报显示：总收入 6.876 亿美元，同比增长 25%；subscription revenue 6.661 亿美元，同比增长 25%；services revenue 2,150 万美元，同比增长 22%。来源：MongoDB Q1 FY2027 SEC exhibit。

这显著高于 Q4 后给出的 Q1 revenue guide。MarketBeat 汇总显示，实际收入 6.8762 亿美元，高于预期 6.6453 亿美元；实际 EPS 1.32 美元，高于共识 1.19 美元。来源：MarketBeat earnings summary。

更关键的是 Atlas。官方财报标题直接写：Atlas revenue up more than 29% year-over-year。电话会转录进一步补充：Atlas 同比增长 29.4%，同比新增收入创纪录达到 1.17 亿美元，Atlas 现在约 20 亿美元 run-rate，并且连续第四个季度同比增长至少 29%。来源：Motley Fool MongoDB Q1 FY2027 transcript。

这直接修复了 Q4 后最危险的市场担忧：Atlas 增速不是突然掉到低 20%，而是在大数基础上继续接近 30%。

MongoDB 还把 FY2027 revenue guide 上调到 29.2 亿至 29.6 亿美元；Q2 revenue guide 给到 7.29 亿至 7.34 亿美元；FY2027 non-GAAP EPS guide 给到 5.95-6.14 美元。来源：MongoDB Q1 FY2027 SEC exhibit。官方标题写得很直接：

Raising full-year fiscal 2027 guidance driven mainly by strength in Atlas.

其他质量指标也很强：RPO 14.586 亿美元，同比增长 88%；cRPO 7.663 亿美元，同比增长 69%；free cash flow 1.975 亿美元，去年同期为 1.059 亿美元；总客户 67,700+，Atlas 客户 66,400+，100K+ ARR 客户 2,895。来源：MongoDB Q1 FY2027 SEC exhibit。

财报后，分析师目标价上调也形成二级催化。例子包括：Morgan Stanley 将目标价从 335 美元上调到 380 美元并维持 Overweight；Needham 将目标价从 300 美元上调到 400 美元并维持 Buy；Barclays 将目标价从 370 美元上调到 387 美元并维持 Overweight；Wedbush 将目标价从 380 美元上调到 390 美元并维持 Outperform；Monness Crespi & Hardt 将目标价从 370 美元上调到 415 美元并维持 Buy。来源：Yahoo / Morgan Stanley syndication、Benzinga / Needham、MarketBeat / Barclays and Monness、MarketBeat / Wedbush。

MarketWatch 显示 2026-06-01 MDB 收盘 403.88 美元，上涨 68.33 美元，涨幅 20.36%，成交量 678 万股，约为 65 日均量 305%。来源：MarketWatch MDB quote。

AI / agent 叙事发生了什么变化

MongoDB Q1 的重要变化是：公司不再只是说“AI 长期有机会”，而是开始给出更具体的 agent backend 路径。

官方业务亮点包括：MongoDB.local London 发布 7 项新能力，用于缩小 AI experiment 到 high-performance production deployment 的差距；这些能力推进 automated retrieval 和 persistent AI agent memory；MongoDB 与 LangChain 正式战略合作，要把 Atlas 变成 production-ready AI agents 的 unified backend，整合 vector search、persistent memory、natural-language querying。来源：MongoDB Q1 FY2027 SEC exhibit。

电话会里，管理层说 AI adoption 正在加速，MCP server usage 显著增长，Voyage customers 环比超过翻倍，vector search adoption 远快于公司整体增长。但管理层也保留了关键限定：本季度结果主要由 core workloads 驱动，AI / agentic workloads 有真实且增长中的 momentum。来源：Motley Fool MongoDB Q1 FY2027 transcript。

这意味着：市场可以开始给 MongoDB AI 期权重新定价，但还不能把 Q1 增长完全归因于 AI。 UBS 的观点也偏克制：结果主要由核心业务驱动，而不是早期 AI demand。来源：Yahoo / UBS syndication。

旧论点应该怎么修正

需要避免的说法：AI 正在打坏 MongoDB 业务。 Q1 revenue +25%、Atlas +29% 以上、RPO +88%、FCF 几乎翻倍、FY guide 上修，都不支持需求崩坏。

也需要避免的说法：MongoDB 不适合 agentic AI。 MongoDB Vector Search、Voyage、Automated Embeddings、LangChain partnership、MongoDB Checkpointer、agent memory/state 叙事都说明它适合一类 agentic workload，尤其是 operational data、persistent memory/state、hybrid retrieval。

但此前关于 Postgres 默认性增强的判断仍然成立。Postgres / Supabase / pgvector 在 AI app starter、MVP、RAG metadata、关系权限、billing 和快速原型中仍是非常强的默认起点。AI 降低 schema / migration 心理成本，也确实削弱了 MongoDB 早期 flexible schema 的相对优势。

更准确的更新框架是“默认权分层”：

  1. AI app starter / MVP / Supabase 模板： Postgres 默认性更强。
  2. 企业 operational data modernization： MongoDB 仍强。
  3. Production agent backend / memory / state / hybrid retrieval： MongoDB 正在争夺新默认权。

所以旧 thesis 不是被推翻，而是要从“MongoDB 默认性受压”更新为：MongoDB 在 AI starter 默认性上受压，但正在 production agent backend 层级反击。

和 Snowflake 的比较有没有变化

此前的 Snowflake / MongoDB 对比结论仍大体成立：Snowflake 的 AI 叙事更容易映射到 product revenue、RPO、NRR 和全年指引；MongoDB 的 AI 叙事仍需要更多季度证明它能转成 Atlas consumption。

Snowflake Q1 FY2027 官方数据很强：product revenue 13.34 亿美元，同比增长 34%；total revenue 13.91 亿美元，同比增长 33%；NRR 126%；RPO 92.1 亿美元，同比增长 38%；FY2027 product revenue guide 从 56.6 亿美元、同比 27%，上调到 58.4 亿美元、同比 31%；Snowflake AI capabilities 使用账户超过 13,600，Snowflake Intelligence accounts 环比翻倍，Cortex Code 超过 7,100 个账户使用。来源：Snowflake FY2027 Q1 SEC exhibit。

差异仍然清楚：

  1. Snowflake 的 AI 更直接变成 enterprise data platform consumption。
  2. MongoDB 的 AI 更像 agentic application backend 的新默认权争夺。
  3. Snowflake 的公开财报已经把 AI 与全年 guide raise 绑定得更紧。
  4. MongoDB Q1 证明的是：核心业务没坏，Atlas 强，AI / agent 早期 traction 足以让市场重新给期权定价。

所以现在不能写成“Snowflake 赢、MongoDB 输”。更准确是：Snowflake 的 AI 货币化确定性仍更强；MongoDB 的业务修复和 agent backend 叙事让它从 Q4 后的折价里显著反弹。

最终判断

今天 MongoDB 暴涨的直接原因是：Q1 FY2027 给了市场想要的 beat-and-raise，并且 Atlas、RPO、FCF、AI / agent 叙事都比 Q4 后预期更强。

此前论点大方向没有错，但语气必须更新：

  1. 短期财务： MongoDB 明显修复，Q4 后悲观预期过度。
  2. 中期产品： MongoDB 不一定赢 AI app starter 默认权，但有机会赢 production agent backend。
  3. 和 Snowflake 对比： Snowflake 的 AI 收入确定性更直接，MongoDB 的 AI 上行期权更偏应用后端和 agent memory/state。
  4. 旧 thesis 状态： 不是推翻，而是从“MongoDB 默认性受压”升级为“默认权分层，MongoDB 正在新层级反击”。

Why Did MongoDB Plunge, Then Surge? Rechecking the MongoDB vs. Snowflake Thesis in the AI Era

MongoDB's latest rally was not a case of the market suddenly forgetting the prior risks. The simpler explanation is that Q1 FY2027 repaired the main concerns that drove the selloff after Q4: Atlas did not break down, full-year guidance was raised, RPO and free cash flow were strong, and the AI / agent narrative moved from a long-term story to early visible traction.

My updated view: the short-term bearish thesis on MongoDB should be downgraded, but the longer-term structural question has not disappeared. MongoDB still faces pressure from Postgres / Supabase / pgvector as the default starting point for many AI apps, but it is now contesting a different layer: the default backend for production agents.

Core View

  1. The March plunge was mainly about expectations and guidance. MongoDB's Q4 FY2026 results were solid, but the initial FY2027 guide implied a growth step-down. Investors worried about Atlas visibility, AI monetization, and broader software-stock valuation pressure.
  2. The latest surge was a classic beat-and-raise. Q1 FY2027 revenue was $687.6 million, up 25% year over year; Atlas grew more than 29%; RPO grew 88%; and MongoDB raised FY2027 revenue guidance to $2.92 billion to $2.96 billion.
  3. It is no longer reasonable to say AI is continuously hurting MongoDB. Q1 was strong evidence that the core business is intact, Atlas remains strong, and AI / agentic workloads are beginning to show more concrete early signals.
  4. The Postgres default-pressure thesis still matters. Postgres / Supabase / pgvector remain powerful default starting points for AI app starters, MVPs, RAG metadata, relational permissions, and billing-heavy application backends.
  5. Snowflake still has more direct AI monetization evidence. Snowflake maps AI adoption more directly into product revenue, RPO, NRR, and full-year guidance. MongoDB's AI upside is more tied to agent memory, state, operational backends, and production agent workloads.

Why the Stock Plunged After the March Earnings Report

MongoDB's Q4 FY2026 quarter was not weak on its face: total revenue was $695.1 million, up 27% year over year; FY2026 full-year revenue was $2.46 billion, up 23%; and Atlas revenue grew 29% in both the fourth quarter and the full year. Source: MongoDB FY2026 Q4 SEC exhibit.

The problem was forward guidance. After Q4, MongoDB guided FY2027 revenue to $2.86 billion to $2.90 billion and Q1 FY2027 revenue to $659 million to $664 million. Source: MongoDB FY2026 Q4 SEC exhibit. Investors saw a company growing 27% in the quarter and 29% in Atlas, but guiding full-year growth to roughly 16%-18%.

Investopedia also framed the decline around disappointing profit guidance rather than a quarterly miss. The article said MDB was down nearly 21%, even though Q4 revenue of $695 million and adjusted EPS of $1.65 both beat expectations; Q1 adjusted EPS guidance of $1.15 to $1.19 came in below the Visible Alpha consensus of $1.21. Source: Investopedia.

The accurate explanation for the March selloff is: strong current quarter, weaker forward setup, AI anxiety across software stocks, and a discount for Atlas visibility. It was not proof that database selection had already shifted away from MongoDB at scale.

Why the Stock Surged This Time

MongoDB's Q1 FY2027 results showed total revenue of $687.6 million, up 25% year over year; subscription revenue of $666.1 million, up 25%; and services revenue of $21.5 million, up 22%. Source: MongoDB Q1 FY2027 SEC exhibit.

That was well above the prior Q1 revenue guide. MarketBeat summarized actual revenue at $687.62 million versus expected revenue of $664.53 million, and actual EPS at $1.32 versus consensus EPS of $1.19. Source: MarketBeat earnings summary.

The most important datapoint was Atlas. The official release says Atlas revenue grew more than 29% year over year. The earnings call transcript adds that Atlas grew 29.4%, delivered a record $117 million of year-over-year dollar growth, reached roughly a $2 billion run-rate, and has now grown at least 29% year over year for four consecutive quarters. Source: Motley Fool MongoDB Q1 FY2027 transcript.

That directly repaired the most dangerous post-Q4 concern: Atlas did not suddenly fall into low-20s growth. It remained near 30% growth at larger scale.

MongoDB also raised FY2027 revenue guidance to $2.92 billion to $2.96 billion; guided Q2 revenue to $729 million to $734 million; and guided FY2027 non-GAAP EPS to $5.95 to $6.14. Source: MongoDB Q1 FY2027 SEC exhibit. The official headline was direct:

Raising full-year fiscal 2027 guidance driven mainly by strength in Atlas.

Other quality indicators were also strong: RPO was $1.4586 billion, up 88% year over year; cRPO was $766.3 million, up 69%; free cash flow was $197.5 million versus $105.9 million a year earlier; total customers were 67,700+; Atlas customers were 66,400+; and customers with more than $100K ARR reached 2,895. Source: MongoDB Q1 FY2027 SEC exhibit.

Analyst target increases added a secondary catalyst. Examples included Morgan Stanley raising its target from $335 to $380 while keeping Overweight; Needham raising its target from $300 to $400 while keeping Buy; Barclays raising its target from $370 to $387 while keeping Overweight; Wedbush raising its target from $380 to $390 while keeping Outperform; and Monness Crespi & Hardt raising its target from $370 to $415 while keeping Buy. Sources: Yahoo / Morgan Stanley syndication, Benzinga / Needham, MarketBeat / Barclays and Monness, MarketBeat / Wedbush.

MarketWatch showed MDB closing at $403.88 on June 1, 2026, up $68.33, or 20.36%, with volume of 6.78 million shares, about 305% of its 65-day average. Source: MarketWatch MDB quote.

What Changed in the AI / Agent Narrative

The important change in Q1 is that MongoDB is no longer just saying “AI is a long-term opportunity.” It is now presenting a more concrete agent-backend path.

MongoDB's official business highlights included seven new platform capabilities announced at MongoDB.local London, aimed at closing the gap between AI experimentation and high-performance production deployment; advancing automated retrieval and persistent AI agent memory; and a formal strategic partnership with LangChain to make Atlas the unified backend for production-ready AI agents across vector search, persistent memory, and natural-language querying. Source: MongoDB Q1 FY2027 SEC exhibit.

On the earnings call, management said AI adoption of MongoDB technologies is accelerating, MCP server usage is growing significantly, Voyage customers more than doubled quarter over quarter, and vector search adoption is far outpacing overall company growth. But management also kept an important qualifier: this quarter's results were driven primarily by core workloads, while AI / agentic workloads are showing real and growing momentum. Source: Motley Fool MongoDB Q1 FY2027 transcript.

That means: the market can start repricing MongoDB's AI option value, but Q1 growth should not be attributed entirely to AI. UBS took a similarly cautious view, saying results were driven mainly by core business rather than early-stage AI demand. Source: Yahoo / UBS syndication.

How the Prior Thesis Should Be Updated

One claim should be avoided: AI is damaging MongoDB's business. Q1 revenue growth of 25%, Atlas growth above 29%, RPO growth of 88%, near-doubling of free cash flow, and raised full-year guidance do not support a demand-collapse thesis.

Another claim should also be avoided: MongoDB is not suitable for agentic AI. MongoDB Vector Search, Voyage, Automated Embeddings, the LangChain partnership, MongoDB Checkpointer, and the agent memory/state narrative all point the other way. MongoDB is suitable for an important class of agentic workloads, especially operational data, persistent memory/state, and hybrid retrieval.

However, the earlier Postgres-default thesis still matters. Postgres / Supabase / pgvector remain powerful default starting points for AI app starters, MVPs, RAG metadata, relational permissions, billing, and fast prototypes. AI also reduces the perceived cost of schema and migration work, which does weaken one of MongoDB's older flexible-schema advantages.

The better updated framework is “default status by layer”:

  1. AI app starter / MVP / Supabase templates: Postgres has the stronger default position.
  2. Enterprise operational data modernization: MongoDB remains strong.
  3. Production agent backend / memory / state / hybrid retrieval: MongoDB is actively contesting a new default layer.

So the old thesis is not overturned. It should be updated from “MongoDB's default status is under pressure” to: MongoDB is under pressure as a default for AI starters, but it is fighting back at the production agent backend layer.

Has the Snowflake Comparison Changed?

The prior Snowflake / MongoDB comparison still mostly holds: Snowflake's AI story maps more directly into product revenue, RPO, NRR, and full-year guidance; MongoDB's AI story still needs more quarters to prove it can become sustained Atlas consumption.

Snowflake's Q1 FY2027 official numbers were strong: product revenue was $1.334 billion, up 34%; total revenue was $1.391 billion, up 33%; NRR was 126%; RPO was $9.21 billion, up 38%; FY2027 product revenue guidance was raised from $5.66 billion, or 27% growth, to $5.84 billion, or 31% growth; more than 13,600 accounts were using Snowflake AI capabilities; Snowflake Intelligence accounts more than doubled quarter over quarter; and Cortex Code was used by more than 7,100 accounts. Source: Snowflake FY2027 Q1 SEC exhibit.

The difference remains clear:

  1. Snowflake's AI story turns more directly into enterprise data platform consumption.
  2. MongoDB's AI story is more about contesting the default backend for agentic applications.
  3. Snowflake has tied AI more tightly to full-year guidance in public financial reporting.
  4. MongoDB's Q1 proved that the core business is not broken, Atlas is strong, and early AI / agent traction is enough for the market to reprice the option value.

So the conclusion should not be “Snowflake wins, MongoDB loses.” A better version is: Snowflake still has more direct AI monetization certainty; MongoDB's business repair and agent-backend narrative helped it rebound sharply from the post-Q4 discount.

Final View

The direct reason MongoDB surged is simple: Q1 FY2027 delivered the beat-and-raise the market wanted, and Atlas, RPO, free cash flow, and the AI / agent narrative were all stronger than post-Q4 expectations.

The earlier thesis was not directionally wrong, but the tone must be updated:

  1. Near-term financials: MongoDB has repaired the story; the post-Q4 pessimism was too aggressive.
  2. Medium-term product position: MongoDB may not win the default position for AI app starters, but it can win the production agent backend layer.
  3. Compared with Snowflake: Snowflake has more direct AI revenue certainty; MongoDB's AI upside is more tied to application backends, agent memory, and state.
  4. Status of the old thesis: it is not overturned; it has evolved from “MongoDB's default status is under pressure” to “default status is splitting by layer, and MongoDB is fighting back in a new layer.”
